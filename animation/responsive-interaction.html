响应式交互

Responsive interaction builds trust with the user and engages them. When a user interacts with an app and beautiful yet perfectly logical things happen, the user feels satisfied—even delighted. It is thoughtful and purposeful, not random, and can be gently whimsical but never distracting. It encourages deeper exploration of an app: what will happen if I touch this? And then this?

在Material Design中，应用应具有快速响应的动效:

  • 触摸，语音，键鼠都是首要考虑的输入方式
  • 尽管UI元素是有型的，但是它们被限制在了设备屏幕之后。视觉及动效线索能够架起用户和设备之间沟通的桥梁，让用户认为自己的输入有效，继续指引用户操作。

迅捷的交互将应用带到了新的高度：不仅仅根据用户请求呈递信息，更赋予沟通真实感。

案例

表层反馈

当接收到输入动作时，系统立刻在交互接触点做出视觉反馈，告知用户输入已确认。例如，指尖接触的触控点，语音输入时的麦克风图标，以及键盘输入的相关区域。这一设计理念来自于墨水：用笔在纸张上写字时，笔与纸的接触点立即出现墨迹。

能够表达这种接触的核心视觉机制是触控涟漪效果。设备会根据触摸事件，立即在触摸点做出视觉表达。可以很好地阐释按压效果和语音输入时音量的变化。

最佳案例

视觉反馈在输入点出现：例如手指和屏幕的接触点。以此点为中心，给予视觉反馈。

物体反馈

除了表面墨水式的反馈，物体本身也能根据交互做出反应。物体因触摸或点击而上升，上升用来表示物体的激活状态。用户可以通过触摸或点击来生成新的物体或者变形已存在的物体。拖动和拨动也能处理物体。

最佳案例

触控原点

当因用户直接交互而生成新物体时，物体表面的扩张动效应该从接触原点开始。

物体因触控上升

当一张卡片或单独的元素被激活时，卡片应该上升，从而暗示其激活状态。

径向效果

所有用户发起的交互行为都有一个中心点：用户通过这个中心点来表达他们的操作意图。围绕用户输入中心点构建清晰的视觉联系，从而使用户确认操作有效。无论是触摸屏幕时的触点，或是语音输入时的麦克风图标，都是输入的中心点。围绕此中心点，根据距离的不同，构建出渐进的涟漪动效。

输入都具备中心点：触控时的触点，语音输入时的麦克风图标，键盘输入时的按键。

动效应从视觉上和中心点相联系。离中心点越近的操作，反馈更明显。反之，离中心点越远，反馈更模糊。这便构成操作的涟漪动效。
